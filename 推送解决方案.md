# 推送代码到 GitHub - 解决方案

遇到网络连接问题，以下是几种解决方案：

## 方案 1：检查网络连接

1. 检查是否能正常访问 https://github.com
2. 如果无法访问，可能需要：
   - 检查网络连接
   - 使用 VPN 或代理
   - 检查防火墙设置

## 方案 2：配置 Git 代理（如果你有代理）

如果你有 HTTP/HTTPS 代理，可以配置：

```powershell
# 设置 HTTP 代理（替换为你的代理地址和端口）
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

# 然后再次推送
git push -u origin main

# 如果不再需要代理，可以取消设置
# git config --global --unset http.proxy
# git config --global --unset https.proxy
```

## 方案 3：使用 SSH 方式（推荐，但需要先配置 SSH key）

### 3.1 检查是否已有 SSH key

```powershell
ls ~/.ssh
```

### 3.2 如果没有，生成 SSH key

```powershell
ssh-keygen -t ed25519 -C "your_email@example.com"
# 按 Enter 使用默认路径
# 设置密码（可选）
```

### 3.3 添加 SSH key 到 GitHub

1. 复制公钥内容：
```powershell
cat ~/.ssh/id_ed25519.pub
```

2. 访问 https://github.com/settings/keys
3. 点击 "New SSH key"
4. 粘贴公钥内容，保存

### 3.4 更改远程仓库地址为 SSH

```powershell
git remote set-url origin git@github.com:Mengxun326/SurviveAUST.github.io.git
git push -u origin main
```

## 方案 4：使用 GitHub Desktop（图形界面）

1. 下载安装 GitHub Desktop：https://desktop.github.com/
2. 登录你的 GitHub 账号
3. 添加本地仓库
4. 点击 "Publish repository" 推送

## 方案 5：使用 Personal Access Token

如果使用 HTTPS 方式，GitHub 现在要求使用 Personal Access Token 而不是密码：

1. 访问：https://github.com/settings/tokens
2. 点击 "Generate new token (classic)"
3. 选择权限：至少勾选 `repo`
4. 生成并复制 token
5. 推送时，用户名输入你的 GitHub 用户名，密码输入 token

## 方案 6：手动上传（临时方案）

如果以上方法都不行，可以：

1. 将项目文件夹压缩为 zip
2. 在 GitHub 仓库页面点击 "uploading an existing file"
3. 上传 zip 文件并解压
4. 或者逐个上传文件

---

**推荐顺序**：先尝试方案 2（配置代理），如果不行再尝试方案 3（SSH），最后考虑方案 4（GitHub Desktop）。
