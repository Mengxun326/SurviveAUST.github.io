# 上传和部署步骤指南

## 第一步：创建 GitHub 仓库

1. **登录 GitHub**
   - 访问 https://github.com/login
   - 使用你的账号登录（用户名：Mengxun326）

2. **创建新仓库**
   - 点击右上角的 "+" 号，选择 "New repository"
   - 或者直接访问：https://github.com/new

3. **填写仓库信息**
   - **Repository name**: `SurviveAUST.github.io`（推荐使用这个名称，可以直接使用 GitHub Pages）
   - **Description**: `安徽理工大学生存指南`
   - **Visibility**: 选择 **Public**（必须公开才能使用免费的 GitHub Pages）
   - **不要勾选** "Add a README file"、"Add .gitignore"、"Choose a license"（因为我们已经有了这些文件）
   - 点击绿色的 **"Create repository"** 按钮

## 第二步：推送代码到 GitHub

在 PowerShell 中执行以下命令（确保你在项目目录 `E:\AUST-CS-Wiki`）：

```powershell
# 1. 检查当前状态
git status

# 2. 如果有未提交的更改，先提交
git add .
git commit -m "完成项目框架搭建"

# 3. 如果远程仓库地址不对，先删除旧的
git remote remove origin

# 4. 添加新的远程仓库（如果仓库名不同，请替换）
git remote add origin https://github.com/Mengxun326/SurviveAUST.github.io.git

# 5. 推送到 GitHub
git push -u origin main
```

**如果遇到网络问题**，可以尝试：
- 使用代理（如果配置了）
- 或者使用 SSH 方式（需要配置 SSH key）

## 第三步：配置 GitHub Pages

1. **进入仓库设置**
   - 在仓库页面，点击 **"Settings"**（设置）标签
   - 在左侧菜单中找到 **"Pages"**

2. **配置部署源**
   - 在 "Source" 部分，选择 **"GitHub Actions"** 作为部署源
   - 点击 **"Save"** 保存

3. **触发部署**
   - 进入 **"Actions"** 标签
   - 如果看到 "Deploy VitePress site to Pages" 工作流，点击它
   - 点击 **"Run workflow"** 按钮
   - 选择 **"main"** 分支
   - 点击绿色的 **"Run workflow"** 按钮

4. **等待部署完成**
   - 部署通常需要 2-5 分钟
   - 可以在 Actions 页面查看部署进度
   - 部署成功后，会显示绿色的 ✓

## 第四步：访问网站

部署完成后，你的网站将在以下地址可用：

**如果仓库名为 `SurviveAUST.github.io`**：
```
https://Mengxun326.github.io/SurviveAUST.github.io/
```

**注意**：如果仓库名不是 `username.github.io` 格式，需要在配置文件中设置 base 路径。

## 第五步：更新配置文件（如果需要）

如果你的仓库名不是 `SurviveAUST.github.io`，需要修改 `docs/.vitepress/config.js`：

```javascript
export default defineConfig({
  // ... 其他配置
  base: '/SurviveAUST.github.io/',  // 改为你的仓库名
  // ...
})
```

然后重新提交和推送：

```powershell
git add docs/.vitepress/config.js
git commit -m "更新 base 路径"
git push
```

## 常见问题解决

### 1. 推送失败 - 网络连接问题

**解决方案**：
- 检查网络连接
- 如果使用代理，配置 Git 代理：
  ```powershell
  git config --global http.proxy http://127.0.0.1:7890
  git config --global https.proxy http://127.0.0.1:7890
  ```
- 或者使用 SSH 方式（需要先配置 SSH key）

### 2. 推送失败 - 认证问题

**解决方案**：
- 使用 Personal Access Token（推荐）
  - 访问：https://github.com/settings/tokens
  - 生成新的 token（需要 repo 权限）
  - 推送时使用 token 作为密码

### 3. GitHub Actions 部署失败

**检查**：
- 查看 Actions 标签中的错误日志
- 确认 `package.json` 中的依赖正确
- 确认 Node.js 版本兼容（需要 18+）

### 4. 页面无法访问

**检查**：
- 确认 GitHub Pages 已启用
- 确认部署成功（查看 Actions）
- 等待几分钟让 DNS 生效
- 检查仓库设置中的 Pages 配置
